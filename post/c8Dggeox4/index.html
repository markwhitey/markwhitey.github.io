初始化一个Git环境

Git init
添加文件到git仓库.

Git add file
可以多次使用,添加文件到暂缓区.

Git commit -m "massage"
完成把暂缓区的所有内容提交到当前分支.


版本管理

Git reset --herd commend ID
head指向的版本就是当前版本,Git允许我们在所有版本的历史内任意穿梭.穿梭前,用git log 查看提交历史,以确认要回退的版本.要重返未来,用git relog 查看命令历史,以便确认未来的版本.


撤销修改

场景一
当你改乱了工作区的某个文件的内容,想直接丢弃工作区的更改时,使用命令git checkout --file

场景二
当你不但改乱了工作区的文件的内容,而且已经提交给了暂缓区,想丢弃修改,分两步:第一步用命令git reset回到上一个版本,也就是场景一,第二步按场景一的方式修改.

git checkout其实是用版本库的版本去替换工作区的版本,无论工作区是是修改还是删除埋在可以一键删除


远程仓库

1.克隆远程仓库

要克隆一个仓库,必须知道仓库的地址,然后使用git clone 命令克隆

git clone git@github:markwhitey.github.io.git

github支持多种协议,包括https,但ssh原生的git协议最快

2.添加远程仓库

git remote add [shortname] [url]

3.查看当前远程仓库

git remote

4.提取远程仓库

  • 从远程仓库下载分支
git fetch

该命令执行后需要执行git merge远程分支到自己的分支

  • 从远程仓库和本地仓库合并
git merge
